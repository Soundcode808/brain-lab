# 전자두뇌연구소 합성 스킬

전자두뇌연구소에서 조사결과/폴더에 쌓인 연구물들을 PROJECT-BRIEF.md에 반영하는 스킬이다.

이 스킬의 핵심 원칙: **조사 결과 = 즉시 실행 신호가 아니다.** 합성 결과를 제안하고, 베스퍼님의 확인을 받은 후에만 적용한다.

---

## 작업 순서

### Step 1: 현재 상태 파악

다음 두 파일을 읽는다:
- `/Users/vesper/Documents/전자두뇌연구소/조사결과/INDEX.md` — 어떤 조사가 있는지 전체 목록
- `/Users/vesper/Documents/전자두뇌연구소/PROJECT-BRIEF.md` — 현재 브리프 상태 (특히 "조사결과 인덱스" 섹션과 "완료" 섹션)

INDEX.md가 없으면 조사결과/ 폴더를 직접 ls로 확인한다.
PROJECT-BRIEF에 이미 반영된 내용과 아직 반영 안 된 조사를 구분한다.

### Step 2: 합성본 읽기

INDEX.md 상단 📌 합성본 섹션에 있는 파일들이 핵심이다. 개별 조사 파일보다 합성본을 우선한다.

아직 PROJECT-BRIEF에 반영되지 않은 합성본을 골라 읽는다.
파일 경로: `/Users/vesper/Documents/전자두뇌연구소/조사결과/[파일명].md`

### Step 3: 핵심 추출

각 합성본에서 PROJECT-BRIEF에 추가할 만한 내용을 추출한다:
- 새로운 결론 또는 권고사항
- 완료 처리해야 할 항목
- "다음 할 일"에 넣어야 할 새 과제
- 조사결과 인덱스 테이블에 추가할 항목
- 실험결과에서 나온 중요한 발견

### Step 4: 변경안 제시 (적용 전 반드시 확인)

추출한 내용을 정리해서 베스퍼님에게 보여준다. 형식:

```
## 반영 제안

### PROJECT-BRIEF "완료" 섹션에 추가
- [추가할 항목]

### PROJECT-BRIEF "다음 할 일" 섹션 변경
- 추가: [새 항목]
- 완료 처리: [완료된 항목]

### 조사결과 인덱스 테이블 추가
| [파일명] | [주제] | [핵심] |

### 기타 변경
- [기타]
```

변경안을 보여준 다음, **반드시 아래 문구로 확인을 받는다. 이 단계를 절대 건너뛰지 않는다:**

> "위 내용으로 전자두뇌연구소 PROJECT-BRIEF.md를 업데이트할까요? (네/아니오)"

베스퍼님이 명시적으로 "응", "네", "해줘", "적용해" 등 승인 의사를 밝히기 전까지는 파일을 수정하지 않는다.

### Step 5: 승인 후 적용

베스퍼님이 승인하면:
1. `/Users/vesper/Documents/전자두뇌연구소/PROJECT-BRIEF.md` 해당 섹션 업데이트
2. 적용 완료 내용 한 줄씩 보고 ("✅ 완료 섹션에 N개 추가됨" 등)

베스퍼님이 수정을 요청하면 변경안을 조정하고 다시 Step 4부터 반복한다.
승인 없이 "알아서 해줘" 같은 말이 나와도 반드시 변경안을 먼저 보여주고 확인을 받는다.

---

## 주의사항

- RAW 폴더는 절대 건드리지 않는다
- PROJECT-BRIEF의 **핵심 철학** 섹션과 **실험 원칙** 섹션은 수정하지 않는다 (베스퍼님이 명시적으로 요청하지 않는 한)
- 합성본이 없으면 개별 조사 파일 중 가장 최신 것을 읽는다
- 이미 PROJECT-BRIEF에 반영된 내용은 중복 추가하지 않는다
- 판단이 어려운 항목은 베스퍼님에게 별도로 질문한다
- 실험결과/의 JSON 파일이나 분석 MD도 필요 시 읽는다

## 파일 경로 기준

```
/Users/vesper/Documents/전자두뇌연구소/
├── PROJECT-BRIEF.md         ← 업데이트 대상
├── 조사결과/
│   ├── INDEX.md             ← 전체 목록 (여기서 시작, 없으면 ls로 확인)
│   ├── 합성-*.md            ← 합성본 (우선순위 높음)
│   └── 조사-*.md            ← 각 조사 파일
└── 실험결과/
    ├── B{번호}-*.md          ← 세션별 실험 분석
    └── *.json               ← 실험 결과
```
