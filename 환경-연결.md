# 환경 연결 — 두 개의 Claude Code 맞추기

> 만든 날: 2026-06-18
> 이 파일 = 컴퓨터(맥)와 웹, 두 곳의 Claude Code가 같은 맥락으로 일하게 하는 안내.

---

## 지금 상황 (한 줄)
**Claude Code가 두 군데서 돈다. 둘 다 같은 깃허브 저장소(`brain-lab`)를 본다.**

| | ① 컴퓨터 (맥 / Claude 앱) | ② 웹 (claude.ai/code) |
|---|---|---|
| 어디서 도나 | 베스퍼님 PC 안 | 클라우드 임시 컨테이너 |
| 파일 | PC의 진짜 파일 | 깃허브에서 복사해 온 것 |
| 저장 | 바로 PC에 남음 | **푸시해야** 깃허브에 남음 |
| 작업 위치 | 보통 `main` 가지 | `claude/...` 가지 → 합쳐서 `main`으로 |

→ 둘은 **별개의 작업 공간**이지만, 깃허브를 가운데 두고 **이어진다.**

---

## 맥락은 어떻게 이어지나 (핵심)
깃허브가 "공동 노트"다. 한쪽이 적어서 **올리면(push)**, 다른 쪽이 **내려받아(pull)** 이어받는다.

```
   맥 Claude Code  ──push──▶  깃허브(brain-lab)  ──pull──▶  웹 Claude Code
        ▲                                                       │
        └───────────────────── pull ◀──── push ─────────────────┘
```

---

## 베스퍼님이 할 일은 딱 두 가지
1. **작업 끝나면 → "올려줘(push)"** 라고 말하기
2. **다른 곳에서 시작할 때 → "최신 받아줘(pull)"** 라고 말하기

나머지(명령어·가지 합치기)는 Claude Code가 알아서 한다.

- **push(푸시)** = 내가 한 작업을 깃허브에 올리기
- **pull(풀)** = 깃허브의 최신 내용을 지금 환경으로 내려받기

---

## 어느 Claude Code든 세션 시작 때 읽을 순서
1. `CLAUDE.md` — 정체성·용어
2. `START-HERE.md` — 행동 규칙
3. `PROJECT-BRIEF.md` — 현재 상태·다음 할 일
4. `세션기록/B{최신번호}.md` — 직전 세션에 무슨 일이 있었나
5. **이 파일** — 지금 어느 환경인지, 동기화 됐는지 확인

→ 이 순서만 지키면 맥이든 웹이든 **같은 맥락**에서 출발한다.

---

## 충돌 안 나게 하는 약속
- **한 번에 한 곳에서만 작업한다.** (맥에서 하다가 웹에서 동시에 X)
- 작업 옮기기 전에 **먼저 push → 옮긴 곳에서 pull.**
- 헷갈리면 그냥 **"지금 양쪽 맞춰줘"** 라고 말하면 Claude Code가 상태를 확인하고 정리해준다.
