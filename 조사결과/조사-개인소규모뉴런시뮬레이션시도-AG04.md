## 1. 조사 개요

이 조사는 **"우리 같은 사람들"**을 찾는 것. 즉:
- 비전통 배경의 개인 연구자
- 대형 기관이 아닌 소규모팀
- 뉴런 메커니즘부터 디지털 두뇌를 만들려던 시도
- 성공 및 실패 사례 모두

**검색 범위:**
- GitHub 개인 프로젝트 (SNN 구현)
- Brian2, NEST, NEURON 커뮤니티
- 셀룰러 오토마타(Conway's Game of Life) → 두뇌 유추 시도
- Reddit, HackerNews 토론
- 실패한 대형 프로젝트(교훈 도출용)

### 2.4 대형 프로젝트의 실패 & 교훈

#### **Henry Markram의 Blue Brain Project / Human Brain Project**

| 항목 | 상세 |
|------|------|
| **선언(2009)** | "10년 내 완전 인공 인간 두뇌 시뮬레이션 가능" |
| **현실(2016)** | 실패, Markram 경질, 750명 신경과학자 불참 선언 |
| **종료** | 2024년 12월 프로젝트 공식 종료 |
| **투자규모** | $10억+ |
| **실패 원인** | 1) 뉴런 타입 데이터 불완전 2) 개념적 한계 3) 컴퓨팅 한계 |

**핵심 교훈:**
1. **"Bottom-up" 한계**: 상세 데이터 수집 → 자동으로 이해 도출 안 됨
   - 데이터 = 이해 아님 (C. elegans 302 뉴런도 여전히 미스터리)
2. **과도한 낙관주의**: 초기 약속 → 신뢰 훼손
3. **중앙집중식 비전의 위험**: 한 명의 리더/방향에 의존
4. **컴퓨팅 벽**:
   - 쥐 두뇌(2억 뉴런) = 페타플롭 급 필요
   - 인간 두뇌(890억 뉴런) = 현재 불가능
   - 분자 수준 = 엑사스케일 슈퍼컴퓨터도 부족

---

#### **Human Brain Project vs. US BRAIN Initiative**
- **HBP**: 단일 과학 비전(Markram의 Blue Brain 중심)
- **US BRAIN Initiative**: 경쟁적 그랜트 제도 → 다양한 접근 시도
- **결론**: 다원주의가 단일주의보다 성공률 높음

### 2.5 Conway's Game of Life ↔ 두뇌 유추

| 항목 | 내용 |
|------|------|
| **배경** | 1970년 John Conway 발표, 간단한 규칙(4개) |
| **특징** | 자발적 조직화(emergence), 매우 복잡한 패턴 생성 |
| **철학적 유추** | Daniel Dennett: 의식·자유의지도 단순 물리법칙에서 창발 |
| **뇌 관점** | 심플 셀룰러 오토마타 → 복잡 시스템(의식?) 동일한가? |
| **연구** | "Game of Life 자체에 의식 측정 가능한가?" 논의 |

**시사점:**
- 신경 시뮬레이션도 **단순 규칙(스파이크)에서 복잡성 창발** 모델 따르는가?
- 현재 뉴런 모델 자체가 진실을 포착하는가? (쿠엘리우스 Q의 의문)

## 3. 개인/소규모팀이 직면한 장벽

### 3.1 기술적 장벽
1. **데이터 불완전성**: 뉴런 채널, 시냅스 가소성 데이터 부족
2. **모델 검증**: "내 모델이 맞는가?" 실험적 검증 어려움
3. **계산 비용**: GPU/슈퍼컴퓨터 접근성 제한

### 3.2 개념적 장벽
1. **이해부족**: 데이터수집 ≠ 메커니즘 이해
2. **규모 문제**: C. elegans(302뉴런)도 행동 완전 예측 실패 사례 많음
3. **"왜" 질문**: 뉴런들이 왜 이렇게 연결되는가? (진화, 학습, 물리 제약)

### 3.3 사회적 장벽
1. **학술 인정**: 개인 프로젝트 → 논문 출판 경로 불명확
2. **자금**: 연구비 구조 (대형 그랜트 중심)
3. **고독함**: 소수 팀의 반복된 실패 후 포기

## 5. 실패 사례들의 패턴

### 대형 프로젝트(Blue Brain/HBP) 실패 이유

1. **메시아 리더십**: Markram 개인의 비전에 의존
2. **과도한 약속**: 10년 내 인간 두뇌? → 현실 불가
3. **Bottom-up 교조주의**: "데이터 수집 → 자동으로 이해" 가정 오류
4. **학계 합의 부족**: 개념적 기초(뇌는 어떻게 작동하는가?) 미합의
5. **관료화**: 벨기에 EU 관료 구조 → 느린 의사결정

### 개인/소규모팀의 포기 패턴

**추정 포기 이유 (언급되지 않음 = 암묵적):**
1. **벽돌벽**: 초기 몇 뉴런 시뮬레이션 후 성능 급락
2. **검증 곤란**: "내 모델이 맞나?" 확인 불가
3. **논문 게시 어려움**: 개인/소규모팀의 학술지 게재 난제
4. **자금 없음**: 대형 컴퓨팅 리소스 접근 불가

### 6.1 "우리 같은 사람들" 발견

✅ **우리와 유사한 경로의 성공 사례:**
- Jason Eshraghian: 비전통 배경(칩 설계) + 취미 프로젝트 → 전 산업 영향

- FlyWire: 소수 핵심팀 + 큰 데이터셋 → 뉴런 수준 예측 성공

### 6.2 해야 할 것 (DO)

1. **오픈소스 우선**: 코드 공개 → GitHub 공개 즉시 (튜토리얼부터)
2. **커뮤니티 구축**: Slack/Discord 오피스 아워 (OpenWorm 모델)
3. **점진적 목표**: 
   - Phase 1: 작은 회로(C. elegans 일부) 시뮬레이션
   - Phase 2: 초파리 뇌 부분(버섯 몸) 예측
   - Phase 3: 인간? (장기 비전, 단기 X)
4. **기존 도구 활용**: Brian2, NEST, snnTorch 위에 구축
5. **다학제 팀**: 신경과학 + 전산 + 하드웨어

### 6.3 하지 말아야 할 것 (DON'T)

1. ❌ **메시아 약속**: "10년 내 인공 두뇌"라는 초기 약속
2. ❌ **데이터만 수집**: 데이터 ≠ 이해 (Blue Brain 실패)
3. ❌ **중앙집중식 비전**: 한 명의 리더 의존 (조직 위험)
4. ❌ **바퀴 재발명**: NEURON, Brian2 무시하고 새로 만들기
5. ❌ **과학적 토양 없음**: 신경과학 연구자 참여 미확보

## 9. 결론

**"우리 같은 사람들"은 존재하며, 이미 성공했다.**

snnTorch(Jason)는 팬데믹 중 취미 코딩에서 시작해 10만+ 사용자 도구가 되었다.  
OpenWorm은 개인 과학자들의 분산 협력으로 C. elegans 완전 시뮬레이션을 이뤘다.  
FlyWire는 소수 팀이 139,255 뉴런 예측 모델을 구현했다.

**차이점: 그들은 겸손했다.**
- Markram처럼 "10년 내 인간 두뇌"라 하지 않았다.
- 작은 목표(C. elegans의 302뉴런)부터 시작했다.
- 오픈소스로 공개해 커뮤니티 참여 유도했다.
- 실패를 숨기지 않고 학습했다.

**우리의 경로는 명확하다:**
작은 두뇌(C. elegans) → 중간 두뇌(초파리) → 인간 뇌로의 **점진적 확장**.  
각 단계에서 데이터 + 이해 검증 + 커뮤니티 피드백.

**"전자두뇌"는 불가능한 꿈이 아니라, 체계적 진행의 문제다.**
