## 01. 뉴런의 수학적 모델들

### 1.1 Hodgkin-Huxley 모델 (HH)

**특징**
- 1952년 개발, 신경과학의 기초 모델
- 생물물리학 기반: 이온 채널 수준의 정확한 표현
- 나트륨(Na+), 칼륨(K+), 누수 채널(Leak) 3가지만으로 행동전위(Action Potential) 재현
- 전압 의존적 이온 채널의 동역학을 비선형 미분방정식으로 모델링

**수식 (핵심)**
```
C_m * dV/dt = -g_Na * m³ * h * (V - E_Na) - g_K * n⁴ * (V - E_K) - g_L * (V - E_L) + I_input
```
- m, n, h: 이온 채널 열림/닫힘 상태 (gating variables)
- 각 변수는 시간과 전압에 따라 변함

**장점**
- 매우 정확한 생물물리학적 모사
- 실험 데이터와 직접 비교 가능
- 개별 이온 채널의 성질 파악 가능

**단점**
- 계산 복잡도 높음 (일반적으로 다른 모델의 50~100배 느림)
- 파라미터 결정 어려움 (매우 많은 생물물리학 상수 필요)
- 대규모 네트워크 시뮬레이션에 부적합
- 일부 채널(AMPA, P2X 등) 다이나믹스는 비선형 Markov 모델로 더 잘 표현됨

**2024-2025 발전**
- 확장된 HH 모델: 인간 P2X, AMPA 수용체 등 리간드 의존 채널을 HH 형식으로 통합 가능하도록 일반화
- 교육용 구현: 확률적(stochastic) 이온 채널 모델 개발로 노이즈 효과 분석

**추천**: 정확도가 중요하고 계산 리소스가 풍부할 때 / 특정 이온 채널 메커니즘 분석 필요 시

### 1.2 Leaky Integrate-and-Fire (LIF) 모델

**특징**
- 가장 단순한 스파이킹 뉴런 모델
- RC 회로(누수 커패시터) 모방: 막 전위가 시간에 따라 지수적으로 감소
- 한 개의 변수(막 전위 V)만 사용
- 임계값(threshold)에 도달하면 스파이크 발생, 즉시 리셋

**수식**
```
C_m * dV/dt = -g_L * (V - E_L) + I_input
if V >= V_threshold:
    V = V_reset
    발생 스파이크 기록
```

**장점**
- 극도로 계산 효율적 (다른 모델의 95% 이상 계산량 감소)
- 매우 해석하기 쉬움 (생물물리 배경 없어도 이해 가능)
- 대규모 네트워크에 이상적 (수십만~수백만 뉴런)
- 수렴 가능한 수학적 분석 가능

**단점**
- 생물학적 타당성 제한적 (이온 채널 세부 사항 무시)
- 복잡한 발화 패턴 표현 불가능 (적응, 버스팅 등)
- 리알 타임 스파이킹만 가능 (서브 임계 진동 같은 현상 미재현)

**사용 사례**
- **초파리 뇌 시뮬레이션 (Nature 2024)**: FlyWire 140,000 뉴런 전체 모델은 LIF 사용
- 이유: 노트북에서 돌릴 수 있는 계산 효율성 + 충분한 정확도

**추천**: **대규모 네트워크, 실시간 처리, 제한된 계산 자원 상황에서 최적 선택**

---

### 1.3 Izhikevich 모델

**특징**
- 2003년 개발, 2개 변수 (V, u)
- LIF의 단순함과 HH의 생물학적 타당성 사이의 중간 지점
- 아래 급수(AHP, after-hyperpolarization) 적응 전류(u) 추가
- 다양한 뉴런 타입의 발화 패턴 표현 가능

**수식**
```
dV/dt = 0.04*V² + 5*V + 140 - u + I
du/dt = a*(b*V - u)
if V >= 30:
    V = c
    u = u + d
```

**장점**
- 2개 변수로 생각보다 복잡한 동역학 표현
- 신경 적응, 버스팅(bursting), 초기 버스트, 이상 발화 등 다양한 패턴 재현
- HH보다 훨씬 빠르면서도 LIF보다 생물학적으로 타당
- 실시간 뉴런 수천 개 시뮬레이션 가능

**단점**
- 파라미터 피팅(fitting)이 데이터에 따라 어려움
- 이온 채널 메커니즘과의 대응 관계가 현상학적(phenomenological)이라 해석 어려움
- 파라미터가 자의적으로 선택될 여지 있음

**추천**: 발화 패턴 다양성이 필요하면서 계산 효율도 원할 때 / 중간 규모 네트워크 (수천~수만 뉴런)

---

### 1.4 Adaptive Exponential Integrate-and-Fire (AdEx) 모델

**특징**
- 2005년 Brette & Gerstner 제안
- LIF에 지수 비선형항 + 적응 전류 추가
- 전압-의존 지수 항이 실험 데이터에서 추출된 값
- 2개 변수 (V, w)

**수식**
```
C_m * dV/dt = -g_L*(V - E_L) + g_L*ΔT*exp((V - V_T)/ΔT) - w + I
τ_w * dw/dt = a*(V - E_L) - w
```

**장점**
- 생물물리학적 근거 있음 (지수 항의 실험적 검증)
- 신경 적응, 초기 버스트, 정규 발화, 불규칙 발화 등 다양한 패턴 표현
- Izhikevich보다 더 생물학적으로 타당
- 파라미터 해석이 명확함

**단점**
- LIF보다는 계산 비용 높음 (하지만 HH보다는 훨씬 효율적)
- 적응 전류의 파라미터가 신경타입에 따라 가변적

**추천**: **생물학적 타당성 + 효율성의 최적 균형 / 신경 적응과 다양한 발화 패턴이 중요할 때**

---

### 1.5 모델별 트레이드오프 정리

| 모델 | 정확도 | 계산속도 | 파라미터 | 발화패턴 | 사용처 |
|------|--------|---------|---------|---------|-------|
| **Hodgkin-Huxley** | 매우높음 | 매우느림 | 매우복잡 | 매우다양 | 특정 채널 분석 |
| **LIF** | 중간 | **매우빠름** | 단순 | 단순 | **대규모 네트워크** |
| **Izhikevich** | 중간~높음 | 빠름 | 중간 | 다양 | 중규모 뉴런군 |
| **AdEx** | 높음 | 빠름 | 중간 | **매우다양** | **추천** |

## 02. Spiking Neural Network (SNN)과 기존 ANN의 차이

### 2.1 근본적 차이

**기존 ANN (Artificial Neural Network)**
- 연속 활성화 값 (0~1 또는 -1~1)
- 시간 개념 없음 (feed-forward, 한 번의 계산)
- 각 뉴런이 입력의 가중합을 활성화 함수로 변환

**SNN (Spiking Neural Network)**
- 이산적 스파이크 이벤트 (0 또는 1)
- **시간축 중요** (수 ms 단위의 타이밍 정보 인코딩)
- 멤브레인 전위가 임계값 도달 시 스파이크 발생
- 시간에 따른 상태 변화 (누적, 적응, 리셋)

### 2.2 SNN의 장점

1. **에너지 효율**: 스파이크 발생할 때만 계산 (event-driven)
   - 뇌: ~20W, AI 모델: ~100~1000W
   - 신경모픽 칩에서 극대화

2. **시간 정보 인코딩**: 신경 스파이킹 시간 자체가 정보
   - 감각 입력의 정밀한 타이밍 정보 보존
   - ANN은 이 정보 손실

3. **온라인 학습**: 스트림 데이터 처리 가능
   - 로봇, 자율주행 같은 실시간 시스템에 유리

### 2.3 SNN의 단점

1. **학습 난제**: 백프롭 알고리즘 적용 불가 (비미분 함수)
   - 대안: STDP (Spike-Timing-Dependent Plasticity)
   - 대안: Surrogate gradient method (2020년대 발전)

2. **하드웨어 종속성**: 신경모픽 칩이 필수가 아니면 오히려 느릴 수 있음
   - GPU/CPU에서는 오버헤드 큼

3. **모델링 복잡도**: 시간 스텝 자체가 하이퍼파라미터

## 03. 신경모픽 하드웨어 (Neuromorphic Chips)

### 3.1 Intel Loihi 2

**하드웨어 사양**
- 128개 신경모픽 코어 (neuromorphic cores)
- 33MB 온칩 SRAM
- 수용 가능: 130,000 개 신경망 구획 + 130,000,000 시냅스
- 2025년 고급 버전 "Kapoho Point": 32개 칩 통합 시스템

**특징**
- 완전히 프로그래머블 (고정 기능 아님)
- **온칩 러닝 (on-chip learning)** 가능: STDP 등 학습 규칙 칩에서 직접 구현
- 비동기 이벤트 기반 (스파이크 발생할 때만 활성)

**성능 (2025 성과)**
- **초파리 뇌 (140K 뉴런, 50M 시냅스) 시뮬레이션에서 350배 고속화**
  - 소프트웨어: ~초당 0.3배 생물 시간 속도
  - Loihi 2: **초당 105배 생물 시간 속도**
- 12개 칩으로 전체 FlyWire 커넥톰 적재 가능

**장점**
- 병렬성 높음 (128개 코어)
- 매우 낮은 전력 소비
- 실시간 처리

**단점**
- 하드웨어 접근 비용 높음
- 프로그래밍 난이도 있음 (전용 도구 필요)

### 3.2 IBM TrueNorth & NorthPole

**TrueNorth (2014)**
- 4,096개 신경시냅스 코어
- 총 1,000,000 디지털 뉴런 + 256,000,000 프로그래머블 시냅스
- **에너지 효율 극대화** (고정 기능 추론에 최적화)
- 비동기 네트워크, 전역 클록 없음

**NorthPole (2024~2025)**
- 차세대 모델
- TrueNorth 기반 개선

**특징**
- 추론(inference)에 매우 강함
- 학습은 오프칩에서 수행 후 칩에 로드

### 3.3 SpiNNaker (University of Manchester)

**사양**
- SpiNNaker 1: 1,000,000개 ARM 모바일 프로세서 핵심
- SpiNNaker 2 (Dresden, 현재 진행중): 5,200,000개 핵심

**특징**
- 마우스 뇌 규모 실시간 시뮬레이션 가능 (약 100만 뉴런)
- 각 핵심이 수백 개 뉴런 + 수백만 시냅스 모델링 가능
- 실시간(biological real-time)

**대안으로서의 위치**
- Loihi보다 더 일반적이고 확장성 높음
- TrueNorth보다 유연성 높음

### 4.2 초파리 (Drosophila, 139,255 뉴런)

#### FlyWire 프로젝트 (2024, Nature 발표)

**데이터 규모**
- 139,255개 뉴런
- **54,500,000개 시냅스**
- 8,453개 뉴런 타입 (기존 미알려 4,581개 신규 분류)
- 전체 뇌 부위 다층 어노테이션

**매핑 방법**
1. 전자현미경(EM) 연속 절편: 초파리 뇌를 ~28,000층으로 슬라이싱
2. AI 자동 세그먼테이션: 각 뉴론 경계 추적
3. 크라우드소싱 검증: Drosophila.ai 플랫폼으로 인간 검증 (몇 만 명)

**시냅스 특성**
- 화학 시냅스(chemical synapses) 기반
- 신경전달물질 예측: 구조적 특징 + 머신러닝
  - 글루타메이트(glutamate, 흥분), GABA(억제) 등 분류
  - 예측 정확도: ~85~90%

#### 초파리 뇌 시뮬레이션 성공 사례 (Nature Computational Science, 2025)

**시뮬레이션 방식**
- 신경 모델: **Leaky Integrate-and-Fire (LIF)**
- 뉴로모픽 구현: Intel Loihi 2 (하드웨어)
- 소프트웨어 구현: Python, Brian 2, PyTorch

**예측 정확도**
- 입력: 미각(taste) + 촉각(touch) 자극
- 출력: 뇌 회로의 활성화 패턴 (calcium imaging으로 검증)
- 결과: **"예측과 실제 뇌 반응이 놀랍도록 정확히 일치"** (Berkeley News)

**구체적 사례**
1. **먹이 개시(Feeding initiation) 회로**
   - 설탕 vs. 물 vs. 소금의 맛 처리 방식 예측
   - 예상: 완전히 다른 회로
   - 실제: 중복되는 회로 많음 → 시뮬레이션이 정확하게 예측

2. **안테나 그루밍(Antennal grooming) 회로**
   - 촉각 자극에 대한 반응 경로 매핑

**성능: 노트북에서 실행 가능한 이유**
- **LIF의 극도의 단순성**: 각 시간 스텝당 ~ 1ms의 계산
- **희소성(sparsity)**: 140K 뉴런 중 언제든 활성은 ~5~10% 수준
- **파이썬 벡터화**: NumPy/PyTorch의 배치 처리
- 전체 시뮬레이션: 노트북에서 생물 시간의 1~10배 속도로 실행

**계산 효율**
| 환경 | 속도 |
|-----|------|
| 소프트웨어 (CPU) | 0.3x 생물시간 |
| 소프트웨어 (CUDA GPU) | 3~5x 생물시간 |
| Intel Loihi 2 | **105x 생물시간** |

**임체된 시뮬레이션 (2025 후속 연구)**
- 시뮬레이션 뇌 + MuJoCo 물리 엔진
- 가상 파리가 실제처럼 걷기, 먹기 등 행동 재현
- 심화 강화학습(Deep RL)과 결합 → 신경모델 개선

#### 신경전달물질 예측의 중요성

**문제**: FlyWire는 구조만 있음 (who-connects-to-whom)
**미해결**: 각 시냅스가 흥분/억제인지, 어떤 신경전달물질인지

**해결책**
- **EM 미세구조**: 시냅스 구조(active zone 크기, vesicle 밀도) 분석
- **머신러닝 모델**: 구조 특징 → 신경전달물질 타입 예측
- **생물 검증**: 면역염색(immunostaining), 현장 하이브리드 분석 등으로 확인

---

### 4.3 생쥐 뇌 (Mice, 매뉴얼리스)

#### MICrONS 프로젝트 (2024-2025)

**규모**
- 1 입방 밀리미터(cubic millimeter) 모두 매핑
- 84,000개 뉴런 (약 200,000개 세포 전체)
- **500,000,000개 시냅스**

**데이터 구성**
1. **전자현미경(EM) 재건**: 28,000개 극박 절편
   - AI + 인간 검증으로 신경 분할(segmentation)
2. **칼슘 이미징(Calcium imaging)**: 75,000개 뉴런의 신경 활동 기록
   - 각성 마우스(awake mouse), 자극 조건
3. **기능적 연결성**: EM 구조와 신경 활동 통합

**2025 성과: "Method of the Year" (Nature Methods)**
- EM 기반 커넥톰 방법론의 성숙도 인정
- 실용화 시작

**인간 뇌로의 확장성 평가**
- 마우스 1mm³ = 뉴런 84,000개
- 인간 뇌 전체 = ~1,400억 개 뉴런
- 필요 EM 볼륨: ~1,600,000mm³ (1.6리터)
- 계산 난제: 막대한 EM 데이터 처리 및 세그먼테이션

## 08. 전자두뇌 구현을 위한 실용적 권고

### 8.1 신경 모델 선택 전략

#### **1단계: 초기 프로토타입 (2026년)**
```
추천: Leaky Integrate-and-Fire (LIF)
```

**이유**
- 초파리 커넥톰 시뮬레이션이 증명한 정확도
- 노트북에서 구현 가능한 계산 효율성
- 수백만 뉴런 규모까지 확장 가능
- 신경생물학 배경 없는 팀도 구현 가능

**구현 방식**
```python
# Brian2 또는 PyTorch로 초파리 뇌 수준 모델 구축
# LIF + ReLU 활성화 + 스파이크 기반 시냅스 가중
# 신경전달물질: 간단한 2분류(흥분/억제)로 시작
```

#### **2단계: 성능/정확도 개선 (2027년)**
```
추천: Adaptive Exponential (AdEx) 또는 Izhikevich
```

**전환 기준**
- 프로토타입 행동이 요구 사항 미달성 시
- 신경 적응, 버스팅 같은 복잡 패턴 필요 시
- 계산 리소스 여유 생겼을 때

**이점**
- LIF보다 생물학적으로 타당
- 여전히 실시간 가능
- 파라미터 해석 가능

#### **3단계: 극도의 정확도 필요시 (2028년 이후)**
```
추천: Hodgkin-Huxley (이온 채널 모델)
```

**조건**
- 특정 신경 질환 시뮬레이션
- 약물 효과 예측
- 별도 GPU 클러스터 투자

### 8.2 커넥톰 데이터 구축 전략

#### **A. 기존 공개 커넥톰 활용**

**1순위: FlyWire 데이터**
- 완성도: 99%+ (크라우드 검증 완료)
- 규모: 140K 뉴런 ✓ 초파리 행동 예측 증명
- 접근: 공개 API (codex.flywire.ai)
- 신경전달물질: 70% 수준 예측 제공
- 타입 어노테이션: 8,453개 분류 + 기능적 역할 주석

**용도**
- 초기 프로토타입 검증
- 신경모델 파라미터 학습
- 행동 예측 테스트 벤치

**비용**: 무료 (공개 데이터)

#### **B. 새 커넥톰 생성 (중기 계획, 2027~2028)**

**옵션 1: 소규모 신경계 매핑**
- 학습 대상: 초파리 특정 뇌 영역 또는 유충
- 방법: EM + 크라우드소싱
- 규모: 1K~10K 뉴런
- 비용: ~수천만원

**옵션 2: 생쥐 뇌 부분 매핑**
- 대상: 해마 또는 후각 피질
- 방법: 기존 MICrONS 기법 활용
- 규모: 10K~100K 뉴런
- 비용: ~수억원

### 8.3 신경전달물질 예측

**중요성**: 커넥톰의 구조만으로는 회로 기능 알 수 없음

**현재 기술**
1. **구조 기반**: EM 미세구조 분석 (활성구역 크기, 소포 밀도)
   - 정확도: ~85~90%
   
2. **머신러닝**: 구조 특징 + 신경 타입 → 신경전달물질 분류
   
3. **생물 검증**: 면역염색, FISH 등으로 확인

**추천**
- 초기: FlyWire 제공 예측 신뢰도 사용
- 중기: 로컬 EM 데이터에 대해 자동 예측 모델 학습
- 장기: 실험적 검증 (높은 비용)

---

### 8.4 행동 검증 프레임워크

**초파리 성공 사례 모방**

1. **입력 자극 정의**
   - 감각 신경 활성화: 시각, 후각, 기계감수용 등
   - 신경조절물질 상태 (도파민, 세로토닌 등)

2. **신경망 시뮬레이션**
   - LIF 모델로 뉴런 활동 계산
   - 출력: 운동 뉴런 스파이킹 패턴

3. **행동 예측**
   - 운동 뉴런 → 근육 활성
   - 물리 엔진(MuJoCo)과 연결
   - 예상 행동 생성

4. **검증**
   - 실제 생물과 비교
   - 칼슘 이미징, 행동 분석 등 멀티모달

**구현 코드 예시**
```python
# 1. 커넥톰 로드
connectome = load_flywire_connectome()

# 2. LIF 뉴런 구성
neurons = create_lif_population(139255)

# 3. 시냅스 연결
synapses = create_synapses_from_connectome(connectome, neurons)

# 4. 자극 입력
sensory_input = apply_stimulus(stimulus_type='taste', intensity=0.5)

# 5. 시뮬레이션
run_simulation(neurons, synapses, sensory_input, duration=1000)  # 1초

# 6. 행동 추론
motor_output = extract_motor_neurons()
behavior = infer_behavior_from_motor(motor_output)

# 7. 검증
compare_with_real_fly(behavior)
```

### 9.2 생물 vs 모델의 갭

**초파리 성공의 조건**
1. 아주 단순한 뇌 (하지만 행동은 복잡)
2. 행동 메커니즘 잘 알려짐 (냄새→먹기, 빛→회피)
3. 회로 수준의 구현으로 충분

**인간으로 확장 시 문제**
- 140억 뉴론은 기술적으로 가능하지만
- 신경조절물질, 글리아, 면역, 호르몬 등 무시한 영향
- **의식, 자아** 같은 고차 기능은 단순 뉴런 회로만으로 가능할까? (미해결 철학 문제)
