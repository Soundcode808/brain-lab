## 0. 핵심 진단 — 우리는 이미 루프의 80%를 갖고 있다

지금 호프(인큐베이터)는 이미 자기개선 루프의 뼈대를 갖췄다:

| 루프 재료 | 우리 현황 | 상태 |
|-----------|----------|------|
| 반복 엔진 | GitHub Actions 매시간 자동 실행 | ✅ 있음 |
| 기억 | `마음상태.json` + `연결상태.npy` + `성장기록.json` | ✅ 있음 |
| 개선 연산자 | 연결 형성/가지치기(form/prune), 감정·판단·예측 | ✅ 있음 |
| A↔B 상호작용 | 신호 주고받기(`A신호.json`/`B신호.json`) | ✅ 있음 (작은 POET) |
| **신호(점수)** | **연결 수(formed) 증가뿐** | 🔴 **부족** |
| **정지·가드레일** | 없음 (이상해져도 계속 감) | 🔴 **부족** |
| **다양성 아카이브** | 없음 (한 상태만 굴림 → 붕괴 위험) | 🔴 **부족** |

**결론: "처음부터 새로 만들기"가 아니라, 빠진 3개(신호·가드레일·아카이브)를 채우면 진짜 자기개선 루프가 된다.**

---

## 1. 가장 큰 문제 — "연결 수 늘리기"는 함정이다

지금 잘됐다/못됐다의 기준이 사실상 **연결 수(formed)** 하나다.
→ 이건 전형적인 **Goodhart 함정**: 시스템이 "그냥 연결을 무한정 늘려서" 점수를 따먹을 수 있다. 그러면 숫자는 오르는데 *진짜로 똑똑해지진 않는다.*

조사 결론과 똑같다: **단일 숫자 점수는 반드시 따먹힌다.** 해법은 **여러 지표를 벡터로** 두고, *"아무 지표도 후퇴 안 하고 + 최소 하나는 개선"* 을 "더 나아졌다"의 정의로 삼는 것.

## 2. MVP 로드맵 — 이 순서대로 쌓는다 (레이어 방식)

START-HERE의 "한 번에 하나씩" 원칙대로, 작은 것부터.

### MVP-0 — 지금 루프를 "안전하게" 만들기 (반나절) 🟢 먼저
무인으로 돌려도 망가지지 않게.
1. **체크포인트 원자적 저장**: `연결상태.npy`를 임시파일에 쓰고 → `os.replace()`로 교체 (중간에 죽어도 원본 안 깨짐).
2. **last_good(마지막 정상본)**: 세션 통계가 정상일 때만(수치 유한 + `avg_active` 정상 범위 + 연결 붕괴 아님) `last_good`으로 승격. 이상하면 다음 실행 때 `last_good` 복원.
3. **cron 변경**: `'0 * * * *'` → `'17 * * * *'` (정시는 깃허브 부하로 *드롭되기 쉬움*).
4. **`concurrency:` 그룹 추가**: 느린 실행이 다음 실행과 겹쳐 상태를 깨는 것 방지.
→ 이것만 해도 "방치해도 안전한" 루프가 된다.

### MVP-1 — 진짜 점수(신호) 정의하기 (가장 중요한 빠진 조각) 🔴 핵심
연결 수 대신/더해, **이미 우리가 계산할 수 있는 4개 지표 벡터**:
1. **자극 구별력** — 다른 입력 패턴에 다르게 반응하나? (B3에서 Jaccard 0.552로 이미 검증) → "전부 같이 켜지는 붕괴"를 직접 막음.
2. **예측 정확도** — `PredictionCore.accuracy()` (이미 ~72% 추적 중). 진짜 "똑똑해짐" 신호 + 예측오류 기반이라 호기심 이론과 맞음.
3. **발화 안정성** — 평균 발화율이 건강한 띠(목표 ~49Hz) 안인가. 폭주·침묵 페널티.
4. **A→B 전달 충실도** — A 신호에 B가 의미 있게 반응하나 (C1→C2 전달 이미 측정).

→ **가중합(하나의 숫자)으로 합치지 말 것** (Goodhart 재발). 미니 Pareto 벡터로 유지.
→ 점수는 훈련이 건드리지 않는 **held-out(숨긴) 탐침**으로 측정해 따먹기 방지.

### MVP-2 — 호기심을 내적 동기로 (이미 90% 됨) 🟡
`PredictionCore`가 이미 놀라움(예측오류)을 계산한다. 이걸 **내적 보상**으로:
- ⚠️ **핵심 함정 "노이즈 TV"**: 예측오류 *자체*를 보상하면, 시스템이 자기가 만든 랜덤(시간기반 씨드)만 멍하니 본다.
- ✅ **해법**: 오류의 **"감소량(학습 진척)"** 을 보상 — 줄어들지 않는 노이즈는 보상 0. 정규화 필수.
→ "새로 *배울 수 있는* 것을 탐색하라"는 원리적 압력이 생김.

### MVP-3 — 작은 아카이브(MAP-Elites 라이트)로 다양성 🟡
한 상태만 굴리지 말고, **3×3=9칸 격자**를 둔다. 축은 이미 계산하는 2개: **활동량 × 연결밀도**.
- 매 세션 결과를 해당 칸에 넣되, 그 칸의 기존 것을 MVP-1 점수로 이길 때만 교체.
- 가끔 *랜덤한* 보관 엘리트에서 다음 세션을 다시 시작.
→ 조기 수렴/붕괴 방지. (v0.2의 "공포 누적 PTSD 붕괴" 같은 사태 때, 아카이브가 붕괴 전 엘리트를 보관해 롤백 가능했을 것.) **두뇌 하나가 아니라 "두뇌 레퍼토리"를 갖게 됨.**

### MVP-4 — 정체되면 탐색 키우기 🟡
`HomeostasisCore`(설정점 복귀)는 이미 가소성 제어 역할. 여기에 **정체 감지**를 `MetacognitionCore`에 추가:
- 성장/점수가 K세션 정체 → 변이(form_prob·자극 다양성) **올림** → 회복되면 다시 내림.
- 이미 `form_prob_boost` 훅이 있으니, 그걸 "효율 기반"에서 "정체 기반"으로 확장.

### MVP-5 (고급, 선택) — AlphaEvolve 패턴 🔵
MVP-1 평가자가 생기면: **OpenEvolve**(AlphaEvolve 오픈 클론)를 `DecisionCore`/`EmotionCore` 계수에 겨냥.
- 진화시킬 부분을 `# EVOLVE-BLOCK-START/END`로 감싸고, `evaluator.py`가 몇 세션 돌려 MVP-1 점수 반환.
- **LLM이 감정→판단의 "규칙 자체"를 진화**시킴. Gemini Flash로 ~100회에 $1~5.
- ⚠️ 평가자가 허술하면 그대로 따먹힌다. 회전하는 held-out 탐침 유지.

## 3. 무엇을 점수로 쓸까 (요약표)

| 분류 | 지표 | 비고 |
|------|------|------|
| ✅ 쓸 것 (싸고 이미 계산 가능) | 예측정확도·예측오류 감소, 자극 구별력(Jaccard), 발화 안정성, A→B 전달, 다양성/아카이브 커버리지 | 벡터로 |
| 🔵 이론적이지만 유망 | 압축 진척("지난주보다 내 활동을 더 잘 예측하나?"), 자유에너지(개념틀로만) | 연구 과제 |
| 🔴 단독 목표 금지 | 연결 수, 활동량, 가중합 단일 숫자 | Goodhart 함정 |

## 4. 처음부터 막아야 할 실패 모드

| 실패 | 증상 | 우리 설계의 방어 |
|------|------|----------------|
| **보상 해킹/Goodhart** | 점수만 오르고 진짜 품질 하락 | 다중지표 벡터 + held-out 탐침 + 가끔 사람이 *어떻게* 점수 땄나 눈으로 확인 |
| **노이즈 TV** | 랜덤만 쳐다봄 | 예측오류 "감소량" 보상 + 정규화 |
| **조기 수렴/붕괴** | 다 똑같이 켜짐, 한 패턴 고착 | 아카이브 + 다양성 압력 |
| **cron 침묵사** | 60일 활동 없으면 워크플로우 자동 비활성 | 자기커밋으로 활동 유지(이미 됨) + 백업 keepalive |
| **비용 폭주** (MVP-5) | LLM 호출 무한 | 반복·예산 상한 |

## 6. 한 줄 결론

> 호프는 이미 "매시간 도는 루프"다. 거기에 **정직한 점수(MVP-1)** 와 **안전장치(MVP-0)** 만 끼우면, "연결을 늘리는 기계"가 **"스스로 더 똑똑해지려 애쓰는 기계"** 로 바뀐다. 나머지(아카이브·호기심·LLM 진화)는 그 위에 레이어로 쌓는다.
